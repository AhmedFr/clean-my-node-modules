# Site i18n + SEO Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Infra tasks are serial (controller-implemented); the four locale-content tasks run as PARALLEL sonnet subagents (disjoint file sets). Spec: `docs/superpowers/specs/2026-07-06-i18n-seo-design.md`.

**Goal:** en/fr/es/de/pt site with translated landing + all 10 blog articles per locale, full hreflang/canonical wiring, per-locale RSS, self-hosted fonts, Lighthouse-100 hardening.

**Architecture:** Twin route trees (`app/*` = en, `app/[locale]/*` = fr/es/de/pt) rendering shared page components; typed TSX dictionaries per locale (rich segments as JSX); blog content in `content/blog/<locale>/` with identical slugs + dates; parity test fails the build on missing translations.

## Global Constraints

- Branch `feat/site-i18n-seo` (stacked on feat/site-tailwind).
- NO em dashes in any language. `node_modules`, `rm -rf`, `npm install` etc. stay as `<code>` tokens untranslated. Prices stay "19 euro(s)"/"29" equivalents per language norms, EUR. Brand voice: direct, technical, no marketing fluff.
- Slugs and dates are IDENTICAL across locales (frontmatter title/description/body translated).
- English URLs do not change. Locale URLs: `/fr`, `/fr/blog`, `/fr/blog/<slug>`, `/fr/blog/rss.xml` (same for es/de/pt). No auto-redirect.
- Every page: self-canonical + `alternates.languages` for all 5 locales + `x-default` -> English URL.
- All gates green per task: typecheck, tests, lint, build.

## Tasks

### Task 1 (controller, serial): i18n infrastructure
- `lib/i18n/`: `locales.ts` (LOCALES=["en","fr","es","de","pt"], DEFAULT_LOCALE, Locale type, isLocale), `Dictionary` type (TSX-capable: rich fields are ReactNode), `dictionaries/en.tsx` (ALL landing+blog-chrome copy extracted verbatim), `getDictionary(locale)` via static imports, `hreflang(path)` helper building the languages map + x-default.
- Components take dict slices as props (Navbar, Hero, StatementBand, Features rows, FeatureGrid, WhyLifecycle, HowItWorks, Download, FinalCta, Footer, blog chrome). Copy lives ONLY in dictionaries afterwards.
- Shared page components `components/pages/HomePage.tsx` + blog page bodies parameterized by locale; `app/[locale]/{page,blog/page,blog/[slug]/page,blog/rss.xml/route}.tsx` + generateStaticParams (fr/es/de/pt; unknown 404); root layout `<html lang>` per tree (en root stays; locale layout sets lang).
- `lib/blog`: locale param (`getAllPosts(locale)` etc. reading `content/blog/<locale>/`), en articles MOVED to `content/blog/en/`; parity test: every locale dir (once present) has the same slug+date set as en.
- Language switcher (footer): links to the equivalent page in each locale.
- Metadata per tree: localized title/description from dict; hreflang everywhere; sitemap emits every URL x locale with `alternates.languages`; JSON-LD gets `inLanguage` + localized description; per-locale RSS.

### Tasks 2-5 (PARALLEL sonnet subagents): locale content fr / es / de / pt
Each agent, for its locale L:
- Write `site/lib/i18n/dictionaries/L.tsx` implementing `Dictionary` exactly (import type from `../i18n.types`), translating `en.tsx`.
- Write 10 articles `site/content/blog/L/<same-slug>.md` translating the English articles: same frontmatter slug/date, translated title/description (<=160 chars) and body; keep commands/code blocks/tables intact; internal links become `/L/blog/<slug>` and `/L`; natural, native-quality prose, not word-for-word.
- Gate: `pnpm typecheck && pnpm test` in `site/` must pass (parity + dict types cover the work).

### Task 6 (parallel with 2-5, sonnet subagent): self-hosted fonts
- JetBrains Mono via `next/font/google`; Cabinet Grotesk (700/800) + General Sans (400/500/600/700) via `next/font/local` with woff2 files downloaded into `site/app/fonts/` (Fontshare ITF FFL permits embedding).
- Wire the three font CSS vars into the `@theme` tokens; delete the `<head>` stylesheet/preconnect links from layout.tsx.
- Gate: build green; fonts render (glyph metrics may shift rendering slightly; visual check by controller).

### Task 7 (controller): a11y/alt audit + integration verify
- aria-hidden on decorative SVGs/mocks, labels on meaningful ones, heading order, link names; Playwright pass on all 5 home+blog pages; switcher round-trip; hreflang/canonical/sitemap/RSS spot-checks; Lighthouse desktop run on /, /blog, one article.

### Task 8 (controller): final review, STATUS, PR
- Whole-branch review subagent (most capable model), fixes, STATUS data block update, PR stacked on feat/site-tailwind (base main after #36 merges).
